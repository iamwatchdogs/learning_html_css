Box Model

The box model is as follows :

What will be the height of the box ?

The Total height = height + Top/Bottom Padding + Top/Bottom Border + Top/Bottom margin

Adding Padding

We can add margin and padding, as follows:

.box-with-marign-and-padding {
    background: rgb(255, 103, 48) url(data:image/png;...) no-repeat;
    margin: 2px 50% 1px 50%;
    padding: 2rem 1rem;
}

Adding Border

We can add border, as follows

.box-with-border {
    background: aqua;
    margin: auto;
    border: 5px solid rgb(255, 0, 34);
}

Border Radius

We can change border Radius, as follows

.box-with-border-radius {
    background: greenyellow;
    border: 5px solid rgb(25, 28, 199);
    border-radius: 25px;
}

Margin Collapse

Margin Collapse tells us that the neighboring box element will be sharing the same margin. If the both neighboring box elements has the same marign level, then the distance between them will ONLY the size of the margin ( thinking that other factors like padding and border are excluded ).

But if there is a difference between the sized of the margin of the neighboring box elementrs then the one which has the greater margin size will be shared among them, just like the below pictures,

Box Sizing

It Determines what out of padding and border is included in elements width and height. basically, this concept works on the bases of inheriting. There might be some cases where the child box element exceeds the size of its parent's size, this might lead to some serious problems. So, this types of problems can be handled by Box Sizing.

Read and try it yourself in MDN Documentation,


Now,Let's take an example :

.parent-box{
    width: 700px;
    height: 400px;
    background: rgb(233, 75, 75);
}
.child-box{
    width: 600px;
    height: 100px;
    padding: 100px;
    border: 50px;
    background: rgb(80, 80, 231);
}


But by adding, ( observe the difference )

.child-box-with-box-sizing {
    box-sizing: border-box;
}
