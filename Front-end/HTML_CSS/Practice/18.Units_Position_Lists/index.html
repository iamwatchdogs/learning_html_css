Units

Always using pixels units is not practice and it does not completely encourage the responsive design. We use different unit for different purpose for even better designning which encourages responsive-ness. These units ar also known as Relative Units.


em : is the unit which sizes the element based on its parent element. for example,

Inside parent ( font-size: 20px; )
Inside child ( font-size: 2em => 2*20px => 40px; )


% : works relatively w.r.t., the size of the parent element. for example,

Inside parent ( font-size: 40px; )
Inside child ( font-size: 50% of 40px => 20px; )


rem : is the unit which is based on the html tag itself. for example,

html sized text ( html { font-size :22px; } )
text ( .rem-example-child { font-size : 1.5 rem => 1.5*22px; } )


vw : is called viewport width. And it's assigns the sizing units based on the users device width. It will assign the value with a relation of the given value and the device width value. for example,

Normal sized text.

text ( .vw-example { width: 50vw => half of device width; } )


vh : is known as viewport height. And it's assigns the sizing units based on the users device height. It will assign the value with a relation of the given value and the device height value. for example,

Normal sized text.

text ( .vh-example { width: 50vh => half of device height; } )

Position

All the content mentioned in the HTML document will be displayed according to the Document Flow. Document flow is something that tells which element should be displayed in which order, and basically it display elements one after another from beginning of the html page to the end of it.

Positioning is a property which allows to position elements w.r.t its surrounding element. Each value give to this property describes the position of the element at a ceratin suitation.

By default all the elements will be of value static, which means it follows the exact same order of document flow to display the html page.

When the the position of any element is static, then we can not use any of the position-related property like top, bottom, left, right and z-index. So, let's see how it behaves with values other than static.

1) Relative

When we assign relative to position property, then we can start using the position property like top, bottom, e.t.c.,

Lets us see an example now,

   ...;

   left: 60px;

2) Absolute

If we assign absolute value to position property then the element of this property will not be consider to be existing in the Document Flow but actually it odes exists.

We can set the position property like top, right, e.t.c., w.r.t., an referencing element this time. This referencing element should the non-static parent element. If its parent is not an static element then it will check whether its parent element is a non-static element or not. This happens until it comes to the <html> tag itself and since there is no parent tag for <html> tag thus it becomes the reference element for setting position.

Lets us see an example now,

Lorem ipsum dolor sit amet consectetur adipisicing elit. Minus consectetur tenetur harum ipsum magnam numquam?

.absolute-example-child {
   ...;
   bottom: 50px;
   left: 60px;
}

List-style Property

The list-styling can be done in short hand as,
