Selectors

Basically the structure of the CSS is as follows:

        Selectors {
            property : value;
        }

Selector(s) plays a key role in applying styling. These selectors are basically the tags/classes/ids which points a part of html contant to apply a specfic type of styling.

There are basically three types of selectors:

  1. Tag selectors
  2. Class Selectors
  3. Id Selectors

Tag selector:

Tag selector(s) is/are the type of selector where we use the html tag directly to style the content within the the tags all over the html page. The styling will be done to every instance of the tag throught the html page. 

        pre {
            color: orange; 
        }

Class selector:

The class selector(s) is/are the type of selectors where the specific styling is applied to the tags specified with the class name within the tag's class attribute. The class styling will overwrite the styling specfic to the tag.

The styling can be applied as follows:

        .classname {
            color: red;
        }

Id selector:

The Id selector(s) is/are the type of selectors where the specific styling is applied to the tags specified with the Id name within the tag's id attribute. The id attribute styling will overwrite the styling specified by the class or tag selectors.

The styling can be applied as follows:

        .classname {
            color: red;
        }

Pseudo Selectors:

Pseudo selectors are basically like extension of selectors based on action. These type of selectors are structured as following: 

        selector:pseudo selector {
            property: value;
        }

For example:

        .pseudo:hover {
            color: blue;
        }

Some of the other pseudo selector which can be applied for specific item on a list. Then, we use some pseudo selectors like first-child, last-child, only-child, nth-child(n), e.t.c
These pseudo selectors will work on the children tags of its parent tag. In this case, the parent tag is <ul> and the children tag is <li>.

Advance Selectors:

We will not be using these type of selectors all the time but its worth knowing about them. The concept of these type of selectors is being more specific about the type of styling we for particular part of html page..
Here, we are going to discuss about 4 Advanced Selectors.

Selector_1 + Selector_2:

This type of selector implementation is used to specify that Apply these stylings to the content which are inside Selector_2 where the Selector_2 is followd by Selector_1.
For example: 

            p + pre {
                color: yellow;
            }

Selector_1 ~ Selector_2:

This type of selector implementation is used to specify that Apply these stylings to the content which are inside Selector_2 where the Selector_2 is followd by Selector_1 and lie within same semantic division..
Note that this selector will be applicable if and only if both of the selectors lie within same semantic division and that's how its different from the selector we discussed before.
For example: 

            p ~ pre {
                color: grey;
            }

Note that if you mentioned both types of implementation metion above and did not divide them semantically then, the styling which was instructed at the end will be applied to both the cases.

Selector_1 > Selector_2:

This type of selector implementation is used to specify that Apply these stylings to the content which is inside All the Selector_2(s) which are followed by Selector_1.
Note that this selector implementation will take place to ALL the direct concurrent child (Selector_2's) which are followed by aleast one Selector_1.

Selector_1 Selector_2:

This type of selector implementation is used to specify that Apply these stylings to the content which is inside All the Selector_2(s) which are followed by Selector_1.
Note that this selector implementation will take place to ALL the directly OR indirectly a concurrent child (Selector_2's) of Selector_1 which are followed by aleast one Selector_1.
