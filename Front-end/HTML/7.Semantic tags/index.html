What is/are Semantic Tag(s) ?

Semantic Tags are the tags which are used to provide meaning to the html document.
Even thought usage of Semantic Tags doesn't reflect anything on the output of the html document,
It's still present. These semantic tags provide the meaning to the html document which is used by
other assistance file for various purposes, like how we specify some html tags in CSS for style.

Some of the Semantic tags are :

  • header
  • footer
  • main
  • navigation
  • aside
  • details
  • summary
  • section
  • article
  • div
  • span

Header, Footer and Main Tags :

Header, footer and main tags are the tags which describes various sections of a html document.
We divide a bulk amount of html code into parts for better orgainzation and to increase the
ease of access various parts of html document.

Basic structure of these semantic tags :


<header id="text">
      content
</header>

<main>
      content
</main>

<footer id="text">
      content
</footer>

Header :
  • Header tag describes the part of html document as head.
  • A html document can contain multiple headers.
  • If you wish to use multiple headers then you must specify an accessing names for each of them.
  • A part of html code is defined as header by wrapping the code as:

    <header id="text">
          content
    </header>
Footer :
  • Footer tag describes the ending part of html document.
  • Just like header tag, a html document can contain multiple footers.
  • If you wish to use multiple footers then you must specify an accessing names for each of them.
  • A part of html code is defined as footer by wrapping the code as:

    <footer id="text">
          content
    </footer>
Main :
  • Main tag describes the the whole main part of html document.
  • Unlike Header & Footer tags, we can not have multiple main tags for a html document.
  • An Single html document contains ONLY A SINGLE main tag.
  • A part of html code is defined as main by wrapping the code as:

    <main>
          content
    </main>

Navigation Tag :

Navigation tag ( <nav> ) is an semantic tag which is used to specify that the part which lies within
these tags has multiple hyperlinks which helps us navigate through html document(s).
Navigation tag has two ways to specify an accessing name and they are,

  • Using attribute called "aria-label",
  • Using attribute called "aria-labelledby"

Using attribute called "aria-label" -

We use this attribute when we are directly specifing the access name of the navigation tag.
Structure looks some like,

<nav aria-label="name">
      content
</nav>


Using attribute called "aria-labelledby" -

We use this attribute when we are depending upon other tag for the access name.
In the present html document, the index is used this attribute to get an access name for it.
Structure looks some like,

<nav aria-labelledby="name">
      <h2 id="name">heading</h2>
      content
</nav>

Here, as we can see. The access name of the naviagtion label is depended upon h2 heading tag.

Aside, details, summary Tags

Aside is an semantic tag which is used to specify a part of html document as a side-note of some other content.
The aside content should be indirectly related to the surrounding content. These are used to placed
some content as a sidebar in a document.

Details is another semantice tag which is used to specify a part of html document i.e., the content palce in
between tags. This semantice tag creates an interaction between the use and the html page, by showing & hiding
the content placed in between these tags. By default, the content placed in these tags are hidden.

The summary tag is just an inter tag of details tag, which concatnates the content place in between the tags to
the small arrow button which is used to show/hide the contents of the details tag.


The syntax of these tag be like,

<aside>
      <details>
            <summary>
                  content
            </summary>
      </details>
</aside>

The output will be like,

Click here to open the hidden content.

This the hidden content.

Section vs article Tags

Section and article tags are the semantic tags which are used to divide the html document for a better orgainzation and
control over the various sections in the body of the html.

Section :

Section tag is an semantic tag which can be used to divide the html document into various section.
We use this tag where we want to divide a part of content which might not have a "complete" meaning on it's own.
Section tag has the attribute called "id" which helps the developers to give a specific accessing name for the section.

A basic structure of a section tag looks like,

<section id="accessing name">
      <content>
</section>

Article :

Just like Section tag, Article tag is an semantic tag which can also be used to divide the html document into various parts.
We use this tag where we want to divide a part of content which has a well-defined meaning on it's own.
Similar to section attribute, Article tag has th attribute called "id" which helps the developers to give a specific accessing
name for the section.

A basic structure of a section tag looks like,

<article id="accessing name">
      <content>
</article>

Div & span Tags

Div and span tags are very similar to the Section & article Tags.
The main difference between Div & span tags and Section & article is that,
div and span tags doesn't have any semantic meaning by themselves, so we need to use CSS or any other external source,
to provide the meaning of these tags. Unlike div & span tags, Section & article Tags have thier
own semantic meaning.

div tag is used to divide a section of html document but the span tag is used for seperating small parts of the html document.
Down below are given their basic structures.

Div tag :

<div id="accessing name">
      <content>
</div>

span tag :

<span>  <content>  </span>
